Artifact 04
Before / After UI Comparisons
Wipro — Status Indicator Redesign
Before
Text-only labels — "Live", "Scheduled", "Failed" with no colour differentiation
All screen group icons identical — impossible to distinguish zones at a glance
No conflict detection — overlapping campaigns discovered only after publish failure
Export button was icon-only — 40% of users missed it entirely in testing
→
After
Colour + icon + label: green circle (Live), amber clock (Scheduled), red cross (Failed)
Zone icons scale with screen count — larger groups more visually prominent on map
Conflict detection inline — warning before publish, must resolve or override to proceed
Export updated to icon + "Export" label — 100% recognition rate in re-test
LTTS — Prescription Flow Redesign
Before — 7 steps
Select patient, navigate to Prescriptions tab, Add Drug, name, dosage, frequency, confirm, then sign on a separate screen
Drug interaction warning appeared only at final submit — too late to act on easily
Digital signature on a separate screen — doctors regularly forgot to complete it
No auto-complete — doctors typed drug names from scratch every single time
→
After — 3 steps
Drug entry inline on consultation screen — zero tab navigation required
Interaction warning shown immediately on drug entry, not at final submit
Digital signature integrated as step 3 on the same screen — cannot be skipped
Auto-complete from common drug list — doctors reported 80% fewer keystrokes

Artifact 05
Usability Testing Findings
Wipro — VisionEDGE · 8 participants · 3 sessions
High severity
Venue map zones unrecognisable
6/8 users could not identify their target screen zone without zooming in. All icons were identical.
Fix: Zone labels + icon scaling by screen count
High severity
Export button missed entirely
5/8 users could not find the export function. Icon-only button blended into the toolbar.
Fix: Icon + "Export" text label added
Medium
No post-publish confirmation
All 8 users immediately refreshed the page after clicking Publish — unsure if it had worked.
Fix: "Going live in ~30 seconds" progress toast
Medium
Conflict detection too late
3/8 users created overlapping campaigns without realising until content failed to play after publish.
Fix: Pre-publish inline conflict warning
Low severity
System view toggle undiscovered
Tech ops users (2/8) searched for device health data before finding the toggle buried in the nav.
Fix: Moved toggle to persistent sidebar
LTTS — Healthcare Platform · 6 clinical staff · 2 sessions
High severity
Signature step skipped consistently
4/6 doctors completed prescriptions without signing — the separate signature screen felt optional.
Fix: Signature integrated inline as step 3
High severity
Nurse touch targets too small
All 3 nurses missed interactive elements on tablets. Buttons were 32x32px — below clinical standards.
Fix: All targets enforced at 44x44px minimum
Medium
Patient history scan slow
Doctors spent 40+ seconds searching flat visit lists for a previous medication. No timeline or search.
Fix: Timeline view + medication filter added
Medium
Auto-save anxiety during consult
5/6 doctors asked "Did it save?" during tasks. No visible indicator — users feared session timeout data loss.
Fix: Persistent "Saved 2 mins ago" indicator

Artifact 07
Challenges I Faced
Wipro — VisionEDGE Platform
1

Challenge

Two completely different users — content managers and technical ops staff — needed to use the same platform but had opposing needs. Content managers wanted simplicity; technical staff needed dense data.

How I solved it

Designed a layered information architecture — a clean default view for content managers, with a single "system view" toggle that revealed device health panels for technical staff. One codebase, two mental models served without either user feeling the other's complexity.

2

Challenge

Engineers pushed back on conflict detection, saying it would slow the publish flow. Product wanted to remove it from the MVP entirely.

How I solved it

Ran a quick usability session — 3 operators, 30 minutes — and showed the exact scenario where campaigns silently fail due to overlap. The evidence made the case better than any design argument. Conflict detection shipped in the first release.

3

Challenge

The UI had to reflect real-time AWS infrastructure states but engineering hadn't finalised API contracts when design work needed to begin.

How I solved it

Documented all 40+ device states upfront in a shared Figma page before a single screen was built. This became the engineering team's own reference document and prevented late-stage redesigns when API contracts changed.

LTTS — Healthcare Platform
1

Challenge

I had zero healthcare domain knowledge when I joined. Clinical terminology, regulatory requirements, and how doctors worked during consultations were completely new to me.

How I solved it

Spent two weeks doing nothing but learning — attended engineering reviews, ran task analysis sessions with doctors and nurses, and built a running documentation log of every workflow. Every design decision had a clinical rationale, not a UX assumption.

2

Challenge

A developer pushed back on my mandatory confirmation dialog before critical device shutdown, saying it added unnecessary friction.

How I solved it

Explained the clinical context clearly — in a regulated medical environment, an accidental shutdown is a patient safety event, not a UX inconvenience. The clinical advisor confirmed this in a joint review. The confirmation step was retained.

3

Challenge

Three completely different roles — doctors, nurses, and admin staff — all used the same platform but had almost nothing in common in their workflows, goals, or usage context.

How I solved it

Designed three separate role-based interfaces instead of one universal UI with tabs. Each had its own primary action hierarchy based on what that role needed to do first — not assumptions about technical level. Clinical validation confirmed the separation before engineering began.
